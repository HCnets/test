# 《23:59 的赛博抽屉》互动空间作品设计文档（V2）喵

## 第一章｜版本记录喵

### 版本目标喵
- V2 的目标是解决“玩法单一、内容深度不足、单局时长偏短”的问题，并把配乐配音与 AI 内容生产纳入统一协议与测试体系喵
- V2 的设计原则是“短回合上头 + 连跑沉浸 + 多支线互补 + 二三级系统进阶 + 可复现的UGC传播闭环”喵

### 版本变更摘要喵
- 新增 3 条差异化玩法支线：支线各自拥有独立资源体系、成长曲线与胜负条件，并与主线通过“兑换、联动事件、跨系统触发”互补而不重复喵
- 所有系统引入二级、三级进阶机制，并补齐“元素融合 + 随机突变 + 连锁反应”的三层深度示例与数值验证口径，确保期望收益偏差≤5%喵
- 新增 5 个“脑洞机制”创意模块，满足“首次出现、跨系统联动、玩家可二次创作”，并给出可落地规则、美术需求、前后端接口字段、异常处理清单喵
- 新增内容深度评估表（玩法/内容/心流三大维度），每个玩法必须达到≥8个子项满分才验收喵
- 输出文档结构重构为六章：版本记录、系统拆分、数值表、接口定义、测试用例、风险评估喵

## 第二章｜系统拆分喵

### 2.1 主线核心循环（保留但强化）喵
- Hub（工位23:59）→ 镜子开场 → 选择抽屉模块 → 输入一句话/一次选择 → 叙事与二选一 → 结算（裂纹/碎片/称号/分享卡/音频）→ 回Hub喵
- V2 强化点是“连跑Run（5局一组）+ 破局目标 + 事件池多级变体 + 音频节奏提示”，把短回合拼成可沉浸长体验喵

### 2.2 主线系统（沿用V1并作为母系统）喵
- 镜子裂纹（MirrorFracture）：世界语气旋钮与事件池权重控制喵
- 三碎片（红豆/糖度/马蹄印）：最小构筑资源，用于解锁、强化与跨系统兑换喵
- 称号（Titles）：风格参数与概率参数，决定“同一句输入在不同称号下会生成不同体验”喵
- 种子口令（Seed）：传播复现器，保证“分享可复现开头、再输入可分叉”喵
- 音频层（BGM/Ambience/SFX/Voice）：用节奏构建沉浸与反馈，让玩家明确感知“进入/选择/结算/升级”喵

### 2.3 新增支线玩法一｜《回声交易所》（社交策略支线）喵
#### 2.3.1 设计目的喵
- 用“社交传播→策略兑换→全局事件”把分享行为游戏化，补上主线缺少的长期策略与社交驱动力喵

#### 2.3.2 独立资源体系喵
- 回声币 EchoCoin：通过“分享卡被二次打开/被领取半价/被二创改写”获得喵
- 共鸣指数 Resonance：由 EchoCoin 投入交易所后产生的指数型成长值，用于解锁更深层的全局事件喵

#### 2.3.3 成长曲线喵
- 1阶（入门）：每次分享生成一个“回声挂牌”，挂牌可被他人领取并给你返还 EchoCoin喵
- 2阶（进阶）：解锁“回声配方”，可把 EchoCoin 兑换为指定事件权重提升（例如提升“合饮故事”出现率）喵
- 3阶（深层）：解锁“镜面ETF”，可以把 EchoCoin 投入全局池，触发“谷雨广播”全服事件（本地可模拟为全局公告），影响所有玩家的今日谷雨签喵

#### 2.3.4 胜负条件喵
- 胜利：在一个Run内将 Resonance 提升至目标阈值并触发一次“谷雨广播”喵
- 失败：在一个Run内 EchoCoin 归零且连续两次分享未被任何人打开（冷启动失败）喵

#### 2.3.5 与主线互补点喵
- 回声交易所不重复主线叙事，而是把“传播行为”变成策略资产，为主线提供事件池调参与破局捷径喵
- 允许用 EchoCoin 兑换主线稀缺资源的“概率提升券”，但不直接给红豆/蹄印，避免破坏主线节奏喵

### 2.4 新增支线玩法二｜《山海里程赛》（探索操作支线）喵
#### 2.4.1 设计目的喵
- 用“地图探索 + 操作节奏 + 风险管理”补上主线缺少的探索与可操作反馈，兼容短视频的爽感循环喵

#### 2.4.2 独立资源体系喵
- 里程 Mileage：每次在模块三或跨模块事件中推进获得喵
- 燃料 Fuel：通过完成微挑战获得（如“节奏跑步”或“选择正确路线”）喵
- 路标 Mark：探索中拾取，用于解锁新地图牌与新音景喵

#### 2.4.3 成长曲线喵
- 1阶（入门）：固定三段路线（雨夜国道/霓虹高架/山海隧道），每段有简单二选一喵
- 2阶（进阶）：解锁“路线拼图”，玩家可用 Mark 拼出自己的路线序列（UGC地图）喵
- 3阶（深层）：解锁“折叠地形”，可触发一次“山海折叠”把路线结构重排，引发连锁事件（见3.3连锁反应）喵

#### 2.4.4 胜负条件喵
- 胜利：在Fuel耗尽前达到目标Mileage并完成终点事件（可触发“逆风终局”音景）喵
- 失败：Fuel归零且当前路线处于“山海惩罚段”，强制回Hub并裂纹+2喵

#### 2.4.5 与主线互补点喵
- 里程赛提供“可操作反馈”和“可探索收集”，并把音频系统的变化变成可感知奖励（新BGM/新Ambience）喵
- Mileage 不直接替代蹄印，而是作为“地图成长”维度，与主线构筑互补喵

### 2.5 新增支线玩法三｜《未寄出审判庭》（叙事策略支线）喵
#### 2.5.1 设计目的喵
- 用“辩论式叙事 + 证据构筑 + 多回合判决”补上主线叙事深度不足的问题，让玩家不仅“看回信”，还要“为选择负责”喵

#### 2.5.2 独立资源体系喵
- 证词 Testimony：从模块二的回信中自动抽取3条关键句作为证词喵
- 证据 Evidence：由玩家选择/改写产生的“证据卡”，每张带权重与副作用喵
- 公信 Credibility：审判庭的血条，决定你能否继续辩论喵

#### 2.5.3 成长曲线喵
- 1阶（入门）：每次审判只进行2回合辩论，给出“寄出/塞回/烧掉”判决倾向喵
- 2阶（进阶）：解锁“证据合成”，可把两张Evidence融合为一张（见3.2元素融合）喵
- 3阶（深层）：解锁“判例连锁”，一次判决会影响下一次审判的法官语气与判词池，形成叙事回声喵

#### 2.5.4 胜负条件喵
- 胜利：在Credibility归零前获得“无罪释放（释怀）”或“有罪成立（承认）”的高分判决之一，并掉落特殊资源“签章Seal”喵
- 失败：Credibility归零且证据矛盾值≥阈值，触发“镜子插话”并裂纹+2喵

#### 2.5.5 与主线互补点喵
- 审判庭不是重复回信，而是把回信变成“可用证据”，让叙事拥有策略层喵
- 产出的签章Seal可以作为“系统级钥匙”，用于解锁脑洞模块与终局破局，不和红豆直冲突喵

### 2.6 五个“脑洞机制”创意模块（首次出现 + 跨系统联动 + 二次创作）喵

#### 模块M1｜《镜像盗梦》喵
- 首次出现点：玩家输入一句话后，系统生成“镜子里的反义梦境”，并允许玩家改写成新的梦境种子喵
- 跨系统联动：镜像梦境会同时影响镜子裂纹与事件池，并可作为回声交易所的挂牌内容喵
- 二次创作：玩家可以把梦境种子分享给别人复现同开场，再由对方改写衍生链条喵
- 规则描述：玩家提交一句“现实句”，系统产出一句“镜像句”，玩家可选择保留/改写，改写会生成新的seed并进入回声池喵
- 美术需求：镜面波纹动效、黑白反转滤镜、梦境卡片模板（标题+两句文本）喵
- 前后端接口字段：request包含{ moduleId:"M1", realLine, mirrorFracture, seedIn? }；response包含{ mirrorLine, seedOut, fractureDelta, echoCoinDelta, shareCard }喵
- 异常处理清单：输入为空→提示默认句并记录；敏感内容→镜像转译为无伤害意象；生成失败→回退到模板梦境库喵

#### 模块M2｜《第二杯拍卖会》喵
- 首次出现点：把“第二杯半价”做成限时拍卖，半价不再免费领取，而是用EchoCoin竞拍喵
- 跨系统联动：拍卖结果会改变模块一的合饮故事权重，同时给里程赛提供Fuel奖励喵
- 二次创作：玩家可自定义“拍卖暗号”（一句话），暗号会变成UGC挑战，别人完成才能参与竞拍喵
- 规则描述：每轮拍卖持续60秒，创建者设暗号与甜度，参与者提交完成凭证并出价，最高者获得“合饮特权”与一段专属猫配音喵
- 美术需求：拍卖倒计时UI、价签动画、猫敲铃音效、成交印章特效喵
- 前后端接口字段：createAuction{ note, sugar, proofRule, duration }；bid{ auctionId, bidEchoCoin, proofPayload }；settle{ auctionId }返回winner与奖励喵
- 异常处理清单：无人出价→自动回流为普通半价；作弊proof→判无效并扣信誉；并发结算→以服务器时间戳为准喵

#### 模块M3｜《山海折叠导航》喵
- 首次出现点：玩家用自然语言生成一张“路标卡”，系统把路标卡映射为地图节点，并可被他人拼进路线喵
- 跨系统联动：路标卡会改变里程赛事件池，同时在模块三触发特殊“山海折叠”连锁反应喵
- 二次创作：玩家可编辑路标卡的标题与一句“路标诗”，形成UGC地图牌库喵
- 规则描述：玩家输入“我想路过什么感觉/景象”，系统产出节点类型（雨/风/霓虹/隧道）与风险值，玩家可保存为路线拼图喵
- 美术需求：地图节点卡（图标+风险星级）、折叠动画（地图重排）、车流/风噪 ambience 切换喵
- 前后端接口字段：genWaypoint{ prompt, mileage, fuel, mirrorFracture }→{ waypointId, type, risk, rewardHint }；saveWaypoint{ waypointId, title, poemLine }喵
- 异常处理清单：过长prompt→截断并回显；风险超界→归一化；保存失败→本地缓存待重试喵

#### 模块M4｜《红豆回声合唱》喵
- 首次出现点：把玩家的判词/金句做成“合唱副歌”，每次收集到红豆就解锁一个声部喵
- 跨系统联动：合唱会改变模块二回信语气与模块三终局BGM选择，并给回声交易所提供高价值挂牌喵
- 二次创作：玩家可以选择3句金句编排成副歌顺序，生成自己的“谷雨合唱谱”分享喵
- 规则描述：收集红豆≥1解锁第1声部（镜子），≥2解锁第2声部（猫），≥3解锁第3声部（同桌），≥4解锁第4声部（导演）喵
- 美术需求：合唱谱界面（四轨道）、播放按钮、波形动画、分享卡（谱面截图风格）喵
- 前后端接口字段：composeChorus{ lines:[...], voices:[...], tempo }→{ chorusId, audioPlan, shareCard }喵
- 异常处理清单：TTS失败→降级为字幕播放；某角色声部不可用→自动替换为镜子声；音频播放中断→支持续播喵

#### 模块M5｜《逆风合约》喵
- 首次出现点：玩家写下一句“逆风宣言”，这句宣言会变成可携带的卡牌，并在关键时刻触发连锁反应喵
- 跨系统联动：合约会消耗/产出headwindCharge，影响称王胜率，并可作为审判庭的关键证据喵
- 二次创作：玩家可分享合约文本作为挑战口令，别人同意签署后会生成“共同合约”产生更强连锁喵
- 规则描述：签署合约时选择代价（裂纹+1或消耗红豆1），获得合约卡；终局触发时若满足条件则触发“连锁：裂纹-2+加冕概率提升”喵
- 美术需求：合约纸张UI、签名动效、盖章SFX、合约卡模板喵
- 前后端接口字段：signContract{ text, costType, state }→{ contractId, card, headwindDelta }；triggerContract{ contractId, context }→{ chainResult, rewards }喵
- 异常处理清单：文本违规→替换为安全合约模板；重复签署→提示并改为升级；触发条件不满足→返回可解释失败原因喵

## 第三章｜数值表喵

### 3.1 数值验证口径（保证平衡误差≤5%）喵
- 本文档采用“期望收益偏差”作为平衡误差指标，定义为：误差=|E_new − E_base| / E_base喵
- E_base 为V1同等时长（同一局/同一Run）下的期望资源收益或期望进度收益喵
- E_new 为引入二级三级机制后，在给定概率与触发条件下的期望收益喵
- 验收标准为误差≤5%，若超过则通过“概率回调、上限封顶、负反馈成本”修正喵

### 3.2 二级三级进阶机制总览（全系统）喵
- 资源系统：基础掉落 → 元素融合（属性化） → 随机突变（分支放大） → 连锁反应（跨系统联动）喵
- 事件系统：单事件 → 事件簇（同主题三连） → 事件回声（影响下一局事件池） → 全局广播（影响今日谷雨签）喵
- 成长系统：称号获得 → 称号天赋（选择1条） → 称号羁绊（两称号联动） → 套装效应（跨模块触发）喵
- 音频系统：固定BGM → 裂纹变体BGM → 结局stinger → 合唱谱动态编排喵

### 3.3 “元素融合 + 随机突变 + 连锁反应”三层机制（以资源合成为例）喵
#### 3.3.1 基础合成（一级）喵
- 输入：任意2个碎片→输出：1个合成物（不带元素）喵
- 基础期望收益 E_base=1.00（按“合成物价值=1”归一化）喵

#### 3.3.2 元素融合（二级）喵
- 规则：合成时根据输入碎片类型确定元素集{雨/风/霓虹/纸}之一，元素只影响事件池与音频，不直接提高价值喵
- 数值：元素不加成价值，E仍为1.00喵

#### 3.3.3 随机突变（三级-1）喵
- 规则：突变概率 p_mut=0.15，突变会把合成物升级为“变体”，价值乘以 m_mut=1.25，但会附带成本 c_mut（裂纹+1或消耗Fuel1）喵
- 期望：E_mut = (1−p_mut)*1.00 + p_mut*1.25 − p_mut*c_costEquivalent喵
- 取等价成本 c_costEquivalent=0.05（将裂纹+1折算为0.05价值损失用于平衡）喵
- 计算：E_mut = 0.85*1.00 + 0.15*1.25 − 0.15*0.05 = 0.85 + 0.1875 − 0.0075 = 1.03喵
- 误差：|1.03−1.00|/1.00=3.0%≤5%喵

#### 3.3.4 连锁反应（三级-2）喵
- 规则：若突变产生“雨元素变体”，则触发连锁概率 p_chain=0.30，连锁会额外掉落1个“路标Mark”或1个“签章Seal”（二选一，按支线缺口补偿）喵
- 折算：将Mark/Seal等价为0.10价值喵
- 期望增量：ΔE_chain = p_mut * p_chain * 0.10 = 0.15*0.30*0.10 = 0.0045喵
- 最终期望：E_new = 1.03 + 0.0045 = 1.0345喵
- 误差：3.45%≤5%喵

### 3.4 数值验证表（示例口径，可扩展到所有系统）喵
- 验证对象：合成系统三级深度引入后，对“单位合成价值”的影响喵
- 基线：E_base=1.0000喵
- 新版：E_new=1.0345喵
- 误差：3.45%喵
- 修正预案：若误差>5%，优先下调p_mut或提高c_costEquivalent，或将连锁掉落改为概率掉落碎片的一部分喵

### 3.5 三条支线的数值边界（避免碾压主线）喵
- 回声交易所：每Run最大可获得EchoCoin上限=120，超过上限进入递减收益（每+10仅计入+5）喵
- 山海里程赛：Fuel上限=8，单局Fuel增益上限=2，避免无限续航喵
- 未寄出审判庭：Credibility初始=5，单回合最大修复=1，避免无限辩护喵

### 3.6 全系统二级/三级进阶的“数值锚点”（逐项给出可验证的E_base）喵
- 镜子裂纹系统E_base：每完成一局，fracture期望变化为0（中性），只由玩家选择偏移，避免系统强行推向某一端喵
- 事件池系统E_base：每局触发1个主事件，期望额外事件数E=0.20（通过低概率“事件簇”提供惊喜但不拖时长）喵
- 称号系统E_base：每Run平均解锁称号期望E=0.40（约2.5个Run解锁1个），避免称号爆炸导致学习成本过高喵
- 资源系统E_base：每局平均获得碎片总量E=1.30（其中主线碎片E=1.00，支线资源折算E=0.30），保证成长可感知但不过快喵
- 音频系统E_base：每局至少1次SFX反馈，voice触发期望E=0.80（避免过度“念台词”造成疲劳）喵

### 4.5 支线接口：山海里程赛喵

- POST /api/road/run/advance：输入routeId与choiceId，输出mileageDelta,fuelDelta,markDelta,events[]喵

### 4.6 支线接口：未寄出审判庭喵

## 第六章｜风险评估喵

### 6.1 风险清单喵
- 内容风险：UGC可能包含隐私、仇恨、极端表达，需要安全转译与quote降刺激策略喵
- 版权风险：BGM必须原创且不可模仿已有旋律，推荐AI生成后人工筛查并留存生成记录喵
- 体验风险：实时生成音频延迟与不稳定，V2采用BGM预置+短句TTS为主以规避喵
- 成本风险：大模型调用频次高，需通过Run内合并请求、模板回退、短句生成减少Token喵
- 平衡风险：多支线可能碾压主线，需上限递减、成本折算与误差≤5%验证喵
- 冷启动风险：回声交易所/拍卖会依赖参与者，可用“机器人占位领取/本地留言池”做演示与冷启动喵

### 6.2 风险应对策略喵
- 安全：输入分级→敏感转译→安全提示→不中断流程→记录safetyFlags用于迭代喵
- 音频：所有音频键值枚举化，模型只选不编，避免路径错误与注入喵
- 平衡：所有收益都以期望值验证，超过5%立即回调概率或提高成本折算喵
- 传播：seed复现保证“同款开头”，二次输入保证“分叉独特”，避免玩家觉得千篇一律喵

### 6.3 交付验收门槛（V2必须达成）喵
- 至少3条支线都可在10分钟演示内跑通一次胜利或失败，并给出可解释的结算与奖励喵
- 所有系统都具备二级三级进阶机制入口，并在一次Run内至少触发一次突变或一次连锁反应喵
- 5个脑洞模块中至少2个可在现场演示“二次创作→他人复现→产生分叉”闭环喵
- 内容深度评估表（见附录执行）中每个玩法达到≥8子项满分才算通过喵
