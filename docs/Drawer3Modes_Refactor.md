# 三抽屉模块彻底重构：三种颠覆玩法模式（小程序版 v0.1）

> 目标：把 A/B/C 三个“抽屉模块”从传统抽屉式叙事交互，重构为 3 种完全不同的玩法范式（不再像抽屉），并在叙事主题/学习成本/视觉冲击力上形成差异化矩阵。
>
> 说明：留存+40%/交互深度+200%是结果指标，无法在离线环境直接保证；本版本提供可验证的数据埋点与 A/B 方案，用于真实数据验证与迭代。
## 0. 约束（小程序现实）
- 触控优先：单手操作、点击/滑动/长按为主
- 弱网/离线：本地统计先跑通，后端版再接云存储/排行/付费
- 性能：中低端稳定 30fps（性能模式降特效与 DPR）

## 1. 三种新模式概览（差异化矩阵）

| 模块 | 新模式名 | 核心机制（非抽屉） | 叙事主题 | 学习成本 | 视觉冲击力 |
|---|---|---|---|---|---|
| A | 呼吸校准（Pulse） | 节律校准 + 容错调参 + 屏息 | 焦虑、边界、自我调节 | 低 | 中（暗角/脉冲/耳语） |
| B | 剪贴室（Cut-up） | 拼贴写作 + 否定反转 + 封蜡提交 | 关系、表达、撤回与承认 | 中 | 中高（拼贴纸张/闪白） |
| C | 逆风走廊（Lane） | 三线换道 + 漂移免伤 + 刹车 | 逃离、选择成本、继续走 | 中 | 高（走廊压迫/撞击反馈） |

## 2. A 模式：呼吸校准（Pulse）
### 2.1 核心玩法
- 画面中心的“心跳/呼吸脉冲”持续跳动
- 玩家点击跟随脉冲获得连击与“平静”
- 左右滑动调整“容错”（更易/更难） → 形成策略：稳定连击 vs 追求高回报

### 2.2 沉浸式流程
1) 黑暗走廊背景 + 轻噪点，提示“先跟上呼吸”
2) 玩家通过节律校准获得“我能慢一点”的自我叙事
3) 连击越高，结算文本越“自洽”（进入后续剧情分支）

### 2.3 创新交互节点（≥3）
- 点击：节律命中/失误（非按钮式）
- 左右滑动：调容错（难度自调）
- 长按：屏息 1 秒（短暂冻结节律，获得额外平静）

### 2.4 视觉风格
- corridor 暗色背景 + 暗角 + 扫描线/颗粒
- 金色脉冲环作为唯一“安全光源”

## 3. B 模式：剪贴室（Cut-up）
### 3.1 核心玩法
- 给 10 个词片，玩家拼成 3~5 个词的句子
- 长按槽位可在词前加“不”（否定反转）
- “封蜡”提交后锁定句子，作为后续剧情输入

### 3.2 沉浸式流程
1) 拼贴纸张的信封背景
2) 玩家“把说不出口的东西剪成可说的形状”
3) 封蜡后进入剧情，选择会影响好感/任务链

### 3.3 创新交互节点（≥3）
- 点词片：入句
- 点槽位：移除对应词
- 长按槽位：否定反转
- 封蜡：提交（可提前结束）

### 3.4 视觉风格
- letter 背景 + 拼贴卡片
- 微闪白作为“说出口的瞬间”

## 4. C 模式：逆风走廊（Lane）
### 4.1 核心玩法
- 三条走廊车道，障碍从远处压近
- 左右滑动换道规避
- 漂移（消耗专注）提供短时间免伤
- 长按刹车减速（降低压迫但减少奖励倾向）

### 4.2 沉浸式流程
1) 走廊压迫感（暗角 + 轻噪点）
2) 玩家不断在“继续走/消耗专注/放慢”中做选择
3) 撞击会触发重要音效与震动，但不阻断剧情

### 4.3 创新交互节点（≥3）
- 左右滑动换道
- 点击“漂移”消耗专注免伤
- 长按刹车
### 4.4 视觉风格
- corridor 背景，障碍红色块带压迫
- 玩家光点为“唯一可控的光”

## 5. 用户路径脑图（文本版）

### 5.1 总览
```
工位Hub
 ├─ 进入Turn(A/B/C)
 │   ├─ 新模式玩法(20~26s)
 │   │   ├─ 关键交互节点(≥3)
 │   │   └─ 结算(奖励/输入句子)
 │   └─ 进入剧情(Story/VN)
 └─ 右侧抽屉
     ├─ 任务链(前置/时间窗/领取)
     └─ 数据看板(漏斗/AB/导出报告)
```

### 5.2 A（呼吸校准）
```
进入A → 跟拍点击 → 调容错/屏息 → 连击达标 → 生成输入句 → 进入剧情
```

### 5.3 B（剪贴室）
```
进入B → 选词入句 → 槽位否定反转 → 封蜡提交 → 生成输入句 → 进入剧情
```

### 5.4 C（逆风走廊）
```
进入C → 换道规避 → 漂移免伤/刹车 → 命中统计 → 生成输入句 → 进入剧情
```

## 6. 技术实现方案（离线版已落地骨架）
- Turn 流程新增 phase：`mode → story → result`
- 新增三模式渲染函数：Pulse/Cut-up/Lane（全屏玩法，不依赖抽屉）
- 结算时将玩法产出转为 `buildTurn(moduleId,input,extra)` 的输入与奖励
- 埋点：`mode_start/mode_action/mode_end/mode_fail`（用于交互深度与留存漏斗）
- 数据看板：设置页入口，可复制 JSON 诊断报告

## 7. A/B 测试计划（离线分桶 + 后端可扩展）
- 分桶：installId hash → A/B
- A：当前强度
- B：更强微恐（暗角/噪点）+ 更紧 QTE/更低容错，但奖励更高
- 指标：
  - 交互深度：`mode_action` / session
  - 完成率：`mode_end` / `mode_start`
  - 失败率：`mode_fail` / `mode_start`
  - 解谜提示率/通关率
## 8. 性能基准测试
- 测试开关：设置 → FPS显示
- 场景：A/B/C 三模式各跑 60 秒，目标：
  - 低端：≥30fps
  - 中端：≥55fps
- 性能模式：降低 DPR/减少特效密度

## 9. 异常场景处理
- 玩法超时：自动进入剧情（记为 fail）
- 设备不支持震动：自动降级
- 设备不支持音频：自动降级
- 用户中途切后台：回到游戏后继续计时或直接结算（后续可做“暂停保护”）
