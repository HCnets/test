# 风险与应对（v0.1）

## 1) “微恐”尺度
- 风险：过度惊吓/引发不适，影响留存
- 应对：全部演出保持克制；提供设置开关（后续补充“微恐强度”）

## 2) 离线环境的社交/排行榜/付费
- 风险：无后端无法闭环
- 应对：离线版先做“分享码/本机榜/助战码”；后端版再接入云存储与支付

## 3) 内容规模与包体
- 风险：5万字与资源增长导致包体膨胀
- 应对：内容走数据包（JSON）+ 热更新；文本压缩；资源复用

## 4) 性能（中低端）
- 风险：动画/特效过多导致掉帧
- 应对：性能模式、DPR上限、特效密度随设备/模式动态下降

## 5) QA 与指标
- 风险：分支多导致逻辑断裂、死路
- 应对：配置校验（连通性/死路/条件矛盾）+ smoke 测试 + 30min 场景脚本
