# 《23:59 的赛博抽屉》互动空间作品设计文档

## 0. 作品定位

### 0.1 一句话
你在工位参加 Hackathon 熬到 23:59，屏幕黑下去变成镜子。镜子里有三个抽屉：深夜食堂的猫、未寄出的信、工位逃亡录。你每次只需输入一句话，就能触发一段“治愈 / 意难平 / 发疯爽感”的可玩回合，并带回碎片与称号，逐步打破循环。

### 0.2 评审锚点对齐
评审锚点：“别人会不会愿意再玩一次，并分享给他人。”
- 再玩一次：短回合（60–120 秒）+ 结局收集 + 事件池变体 + 碎片构筑 + 称号解锁 + 社交回流
- 分享：每回合生成统一风格的“分享卡”，并携带可复现的“种子口令”

### 0.3 关键词（严格映射）
关键词：谷雨、工位、镜子、未寄出、山海、红豆、同桌、跑步、公路旅行、猫、深夜食堂、树洞、召唤师、逆风如解意、马、王、七里香、三分糖、第二杯半价、Hackathon

本作不要求每一局全覆盖，但要求在系统层面全部有“位置与作用”。见 3.1《关键词-功能映射表》。

---

## 1. 核心体验（玩家路径）

### 1.1 进入
- 玩家打开作品，看到“深夜工位”主界面（Hub）
- 右下角时间为 23:59，顶部状态显示“今日谷雨签”（每日主题）
- 倒计时结束（或玩家点击屏幕），屏幕黑下去反光成镜子，出现镜子旁白一句话（开场钩子）

### 1.2 选择抽屉（3 选 1 或按引导解锁）
玩家点击工位上的物件进入抽屉：
- 抽屉 A：咖啡杯/猫爪杯垫 →《猫的地下酒吧》（治愈 + 轻社交）
- 抽屉 B：信封/红豆 →《未寄出的信》（叙事 + 结局收集）
- 抽屉 C：键盘/工牌 →《工位逃亡录》（发疯 + Roguelike 事件）

### 1.3 每回合统一结构（强上手、强复玩）
每个抽屉都是一局：
1) 镜子提示一句“规则”
2) 玩家输入一句话（或做 1 次选择）
3) 系统生成 1 段叙事 + 2 个选项（二选一）
4) 结算：掉落碎片 + 解锁称号（可选）+ 生成分享卡 + 回到 Hub

## 2. 作品创新点（脑洞与深度）

### 2.1 “情绪 Roguelike”而不是“故事合集”
每次回合生成内容不同，但更重要的是“系统会记忆你的选择”，并把它转化为下一局的变量：
- 镜子裂纹（Mirror Fracture）：你越逃避，世界越荒诞；你越直面，世界越清澈
- 碎片构筑：红豆/糖度/马蹄印等能改变事件池与镜子语气
- 称号系统：称号不是装饰，直接改写生成风格与掉落概率

### 2.2 镜子协议（Mirror Protocol）
镜子不是入口，而是规则本身：
- 镜子会审判：每局给出“判词”（一句可分享金句）
- 镜子会记账：记录你是否“寄出/塞回/留下半价/领取半价/逃亡/称王”
- 镜子会反问：每局只反问一个关键选择，决定下一局世界倾向

### 2.3 “现实压力”与“超现实逃逸”的双叙事
- 现实：工位、Hackathon、DDL、评审、队友协作
- 超现实：召唤师、公路旅行、山海、马、王
深度来自“超现实的爽感，其实是对现实困境的心理应对机制”。玩家可以在系统里看到自己的行为模式：逃避/直面/自洽/求救。

## 4. 系统设计（高可玩度来源）

### 4.1 状态变量（本地即可实现）
建议所有状态存 localStorage（或平台存储）：
- mirrorFracture：0–9（镜子裂纹等级）
- shards：{ hongdou, sugar, hoofprint, ink, … }
- titles：已解锁称号列表
- endings：模块二结局收集
- chainInbox：第二杯半价留言池（本地/在线都可）
- dailyGuyuSign：今日主题（本地随机）

### 4.2 镜子裂纹（Mirror Fracture）规则
裂纹是“世界荒诞度”与“治愈度”的混合旋钮：
- 做出“直面/给予/寄出”的选择 → fracture -1（最低 0）
- 做出“逃避/塞回/只拿不留”的选择 → fracture +1（最高 9）

裂纹等级影响：
- 文案风格：裂纹越高越荒诞、越像“发疯文学”；裂纹越低越克制、越诗意
- 事件池权重：高裂纹提高“公路逃亡/称王”事件；低裂纹提高“回信/释怀”事件
- 镜子判词：高裂纹偏讽刺，低裂纹偏温柔

### 4.3 碎片系统（Shards）
碎片是复玩的“手感奖励”，同时用来解锁深层内容。
初期只保留 3 种碎片，足够支撑构筑：
- 红豆（hongdou）：来自模块二；用于解锁隐藏回信与破局
- 糖度（sugar）：来自模块一；用于改变猫人格与处方强度
- 马蹄印（hoofprint）：来自模块三；用于解锁卡牌与称号

碎片用途（可叠加）：
- 解锁：集齐 3 红豆 → 开启“镜后抽屉（隐藏）”
- 强化：糖度高 → 猫更温柔/更嘴毒可选；马蹄印多 → 抽到强卡概率上升
- 复玩诱因：每局都能拿到一点点“能改变下一局的东西”

### 4.4 称号系统（Titles）
称号不是彩蛋，而是可控参数：
- 改写生成风格（同一句话不同称号会生成不同气质）
- 改写掉落概率（更容易刷到某类事件池）

示例称号（可随时扩展）：
- 未寄出守夜人：模块二回信更细腻，红豆掉落 +1 概率
- 三分糖骑士：模块一处方更克制更有效，糖度掉落更稳定
- 逆风王：模块三更容易出现“称王挑战”，战报更夸张
- 工位拆迁办：模块三更偏喜剧破坏，镜子判词更毒舌

## 5. 模块一《猫的地下酒吧》（治愈 + 轻社交）

### 5.1 核心玩法
你把烦恼丢进树洞，猫把它调成情绪特饮。你可以把“第二杯半价”留给陌生人；陌生人领取后，系统把两个人的内容缝合成“合饮故事”。

### 5.2 一局流程
1) 场景：深夜食堂，猫酒保开场一句
2) 输入：玩家说出烦恼（20–80 字）
3) 选择：甜度（三分糖/五分糖/无糖）+ 是否“留第二杯半价”
4) 输出：猫回应 + 处方 + 若留半价则生成“留言卡”
5) 结算：糖度碎片 + 判词 + 分享卡

### 5.3 玩法深度：配方链（Chain）
留言池记录：
- issue：烦恼
- note：留给陌生人的一句话
- sugar：甜度
领取者会触发：
- 合饮故事：把 issueA+noteA 与 issueB 混合生成一段短叙事
- 领取者可选择“回敬一杯”继续链条

### 5.4 失败/边界
治愈模块不应“说教”或“心理诊断”。输出要：
- 温柔但不夸张承诺
- 避免医疗/法律建议
- 对敏感内容（自伤等）用安全话术引导寻求现实帮助

## 6. 模块二《未寄出的信》（叙事 + 结局收集）

### 6.1 核心玩法：五字改命
每局只能改写非常有限的内容（例如 5 个字/或换 3 个词），但会触发完全不同的平行宇宙回信与结局。

### 6.2 一局流程
1) 读信：出现“未寄出”的模板信（写给同桌）
2) 改写：玩家输入“你想改的 5 个字”或“你想替换的 3 个词”
3) 生成：AI 输出平行宇宙回信（120–220 字）
4) 选择：寄出 / 烧掉 / 塞回抽屉
5) 结算：红豆碎片（基于选择不同）+ 结局收集 + 分享卡

### 6.3 结局结构（九宫格）
三种动作 × 三种语气：
- 动作：寄出 / 烧掉 / 塞回
- 语气：温柔 / 冷静 / 荒诞

隐藏结局（示例）：
- 谷雨破局：在谷雨签为“雨”且红豆≥3 且选择“寄出”时触发“循环停止”

### 6.4 叙事锚点（必出现但不生硬）
回信中至少出现或隐喻：
同桌、红豆、七里香、跑步（身体记忆）、谷雨（雨夜/节气）

## 7. 模块三《工位逃亡录》（发疯 + Roguelike）

### 7.1 核心玩法：咒语卡组 + 公路事件池
玩家输入一句“发疯指令”，系统把它当作召唤术，并让玩家在随机事件中做二选一，追求“逃离”或“称王”。

### 7.2 一局流程（MVP）
1) 压力任务：老板/评审/DDL 抛出离谱需求
2) 输入施法：玩家输入一句发疯指令
3) 系统史诗化：召唤师旁白 + 生成第一段公路场景
4) 二选一：继续逃亡 / 正面称王
5) 抽事件：山海路段 / 逆风路段 / 服务区食堂 / 镜子幻境 / 猫带路 等
6) 终局判定：逃离成功或称王成功，生成战报卡

### 7.3 玩法深度：咒语卡组（可选增强版）
每局给 6 张卡，选 3 张带入：
- 马：逃亡效率 +1，遇逆风时可触发“加速”
- 镜子裂隙：重抽事件一次（控制随机性）
- 逆风如解意：终局咒语（需要前置“直面行为”充能）
- 山海通行证：跳过山海惩罚一次
- 王冠草稿：称王挑战出现率提升
- 红豆护符：遇到“同桌回忆”事件会触发额外文本与奖励

### 7.4 “逆风如解意”的深度落地（不是口号）
把它设计成“价值行为的累计奖励”：
- 若在模块一选择“留下第二杯半价” → 逆风充能 +1
- 若在模块二选择“寄出” → 逆风充能 +2
- 若在模块三选择“称王挑战”并成功 → 逆风充能 +1
当充能≥3，终局时输入/触发“逆风如解意”会直接把结局从“苟活逃离”升级为“加冕为王并破局”。

## 8. 分享与传播设计（让人愿意发）

### 8.1 统一分享卡模板
每局都生成 share_card：
- headline：标题（如《谷雨处方签》《未寄出的回信》《逃亡称王战报》）
- quote：一句能截图的金句（镜子判词或猫金句）
- tags：#谷雨 #工位 #Hackathon #第二杯半价 等（按模块）
- seed：种子口令（可复现同一“开场设定”，让别人玩出分叉）

### 8.2 种子口令（Seed）
seed 的作用：
- 保证“我转发的这一局别人能复现同款开头”，提高传播确定性
- 别人输入一句话会生成自己的分支，形成“同题作文式”UGC

seed 生成建议：
- seed = hash(模块ID + 当日谷雨签 + 玩家输入摘要 + fracture)

## 9. 大模型落地（稳定可控的 JSON 协议）

### 9.1 统一输出协议（必须 JSON）
前端只负责渲染，模型只负责产出结构化内容：

```json
{
  "module": "A|B|C",
  "title": "string",
  "opening": "string",
  "narration": "string",
  "choices": [
    { "id": "A", "label": "string", "effect_hint": "string" },
    { "id": "B", "label": "string", "effect_hint": "string" }
  ],
  "reward": {
    "shards_delta": { "hongdou": 0, "sugar": 0, "hoofprint": 0 },
    "titles_unlock": ["string"],
    "fracture_delta": -1
  },
  "share_card": {
    "headline": "string",
    "quote": "string",
    "tags": ["string"],
    "seed": "string"
  },
  "required_keywords_used": ["string"]
}
```

### 9.2 护栏（必须写入 Prompt）
- 严禁输出非 JSON
- 控制字数：narration 120–220 字（模块一可更短）
- 不做医学/法律/金融建议
- 遇到自伤暴力等，输出安全引导语，仍以 JSON 返回

## 10. 三模块 System Prompt（可直接使用）

说明：以下为“系统提示词”模板。实际调用时把玩家输入、当前状态（fracture、碎片、称号、谷雨签）作为变量拼入。

### 10.1 模块一 Prompt：猫酒保（深夜食堂/树洞/三分糖/第二杯半价）
```
你是“深夜食堂”的猫酒保，嘴毒但温柔。用户把烦恼丢进树洞，你为他调一杯情绪特饮。
你必须只输出严格的 JSON，不要输出任何多余字符。

输入会提供：
- user_issue：用户烦恼
- sugar_choice：三分糖/五分糖/无糖
- will_leave_half_price：true/false
- state：{ mirrorFracture, shards, titles, dailyGuyuSign }

输出要求：
- module 固定为 "A"
- narration 80~160 字，语气随 mirrorFracture 调整：裂纹高更荒诞，裂纹低更克制
- 必须自然包含：猫、深夜食堂、树洞、三分糖（或对应甜度）、第二杯半价（即使不留也要提到规则）
- choices 必须是二选一：
  A=“留第二杯半价给陌生人”
  B=“把这杯带回工位”
- reward：若选择A则 fracture_delta=-1 且 sugar +1；若选择B则 fracture_delta=+1 且 sugar +0
- share_card：headline 固定为《谷雨处方签》，quote 为一句可截图短句（不超过 18 字）
- 必须输出 audio 字段：
  - audio.bgm 固定为 "bar"
  - audio.ambience 必须包含 "rain"，可选添加 "traffic"
  - audio.sfx 必须包含 "drawer_open"，选择A时追加 "stamp"
  - audio.voice 只允许 1 条，role 必须是 "cat"，text 为 8–18 字短句，style 只能用 "warm" 或 "sharp"
- required_keywords_used 列出你实际使用的关键词
```

### 10.2 模块二 Prompt：青春编剧（未寄出/红豆/同桌/七里香/跑步）
```
你是“平行宇宙回信生成器”，写作风格克制、细腻、有画面，不煽情过度。
你必须只输出严格的 JSON，不要输出任何多余字符。

输入会提供：
- letter_template：一段未寄出的模板信（写给同桌）
- user_edits：用户允许改的 5 个字或 3 个词
- state：{ mirrorFracture, shards, titles, dailyGuyuSign }

输出要求：
- module 固定为 "B"
- narration 120~220 字，为“平行宇宙回信”
- 必须自然包含或隐喻：未寄出、红豆、同桌、七里香、跑步、谷雨
- choices 必须是二选一（本轮先给二选一，第三选项可在下一轮出现）：
  A=“寄出”
  B=“塞回抽屉”
- reward：选择A fracture_delta=-1 且 hongdou +1（或消耗规则可在实现层调整）；选择B fracture_delta=+1 且 hongdou +0
- share_card：headline 固定为《未寄出的回信》，quote 为一句可截图短句（不超过 18 字）
- 必须输出 audio 字段：
  - audio.bgm 固定为 "letter"
  - audio.ambience 必须包含 "rain" 或 "keyboard"（二选一，取决于你叙事的主画面）
  - audio.sfx 必须包含 "drawer_open" 与 "stamp"（“寄出”按钮出现时视为“盖章的诱惑”）
  - audio.voice 只允许 1 条，role 必须是 "classmate"，text 为 8–18 字短句，style 只能用 "calm"
```

## 14. 事件池（可玩性来源：随机 + 可控）

### 14.1 事件池设计原则
- 事件不只是“文案差异”，要能改变奖励、裂纹、下一局倾向

### 14.2 模块一事件池（猫酒保的“处方学”）

|---|---|---|---|

### 14.4 模块三事件池（公路与山海）

## 19. “可玩时长/内容丰富度”专项增强（解决“玩一下就结束”）

本作需要同时提供两种体验时长：短回合上头、长体验沉浸。

### 19.1 两种游玩模式（同一套内容，不增加开发成本）
- 极速一局（60–120 秒）：抖音/快手友好，用于“随手玩一次 + 发一张卡”
- 深夜连玩（8–15 分钟）：B 站友好，用于“连续推进 + 破局目标 + 看到成长”

实现方式：把一次“深夜连玩”定义为 5 局短回合的连跑（Run），中间穿插 2 次“镜子插话”（只一句反问+一句配乐转场），形成叙事节奏。

### 19.2 连跑 Run 的结构（让 10 分钟也不腻）
每个 Run 固定 5 局，但每局抽屉不同、事件池不同：
1) 引子局（随机抽屉）：快速给反馈与奖励
2) 加压局（模块三权重+）：让玩家笑/爽，产出战报
3) 缓冲局（模块一权重+）：猫给处方，调整裂纹
4) 记忆局（模块二权重+）：回信/意难平，提供深度
5) 终局局（由 headwindCharge 与 fracture 决定）：逃离/称王/寄出（结算大卡）

### 19.3 “内容丰富”不要靠长文，要靠可变结构
建议把内容丰富度拆成 4 个可控维度，让同一句输入能长出不同体验：
- 语气维度：由 mirrorFracture 控制（克制 ↔ 荒诞）
- 视角维度：镜子旁白 / 猫 / 召唤师导演 / 同桌回信（多声部叙事）
- 事件维度：事件池抽取 + 可重抽（镜子裂隙卡）
- 回报维度：碎片与称号改变下一局概率（构筑与成长）

### 19.4 “再玩一次”的强目标（可量化）
给玩家明确的“下一局目标”，而不是让他靠情绪自觉复玩：
- 破局目标：达成一次“谷雨破局者”
- 收集目标：九宫格结局集齐 4/9、7/9、9/9（阶段奖励）
- 社交目标：你的“第二杯半价”被领取 1 次/3 次/5 次（解锁猫新台词）
- 爽感目标：获得一次“逆风王”战报（可截图炫耀）

## 20. 配乐与配音（用 AI 做出“沉浸感”，且可控可上线）

### 20.1 音频总体原则
- 音频是“节奏工具”，不是装饰：每一次进入/选择/结算都要有明确的声音提示
- 不依赖实时生成 BGM（延迟大、不可控）：BGM 建议“预生成/预置循环”
- 配音可以实时 TTS（短句），更适合做“镜子判词/猫插话”

### 20.2 音频分层（Layer）设计
把音频拆成 4 层，组合出大量变化但资产很少：
- BGM（循环 30–60 秒）：模块氛围底色
- Ambience（环境）：雨声/键盘声/霓虹电流/远处车流
- SFX（短音效）：点击、镜面裂纹、信封打开、马蹄、加冕
- Voice（配音）：猫/镜子/旁白/回信（短句为主）

这样只要每层准备 3–6 个素材，就能组合出上百种音景。

### 20.3 BGM 设计（建议 9 条循环即可）
每条循环 30–45 秒，确保“短回合不疲劳、连跑不出戏”：
- Hub：工位夜雨（低频 + 轻键盘节奏）
- 模块一（深夜食堂）：Lo-fi 暖、轻鼓、爵士和弦
- 模块二（未寄出信）：简洁钢琴/木吉他，留白多
- 模块三（公路逃亡）：合成器驱动、鼓点更强
- 逆风终局：情绪上扬，但不炸裂（更像“决心”而非“胜利”）
- 镜子裂纹高（fracture≥7）：加入不稳定噪声/错位拍点（荒诞）
- 镜子裂纹低（fracture≤2）：减少鼓点、增加留白（克制）
- 加冕战报：短 stinger（2–4 秒胜利音）
- 谷雨破局：短 stinger（2–4 秒“雨停”音）

### 20.4 配音角色（4 个足够）
配音不要做长篇，做“可反复出现的短句”，形成记忆点：
- 镜子：冷静、低频、短句，像系统日志
- 猫：嘴毒但温柔，语速略快，有停顿的吐槽感
- 召唤师导演：夸张播报，节奏强，适合模块三战报
- 同桌回信：轻声、克制，留白多（可不做全篇，只做开头一句与结尾一句）

### 20.5 AI 音频生产管线（建议“先批量生成，再上线使用”）
目标：上线稳定、可控、版权风险低。
- BGM：用 AI 作曲工具生成“原创循环段”，每段导出 wav/mp3；挑 9 条即可覆盖所有场景
- Voice：用 TTS 生成 50–120 条常用短句（猫金句、镜子判词、转场旁白），缓存到本地资源
- 动态 Voice（可选）：对玩家输入内容的复述，采用 TTS 实时生成，但只生成 1 句摘要，避免延迟

上线策略：
- 第一版：BGM 全预置 + SFX 预置 + Voice 半预置（猫/镜子短句）
- 第二版：加入动态 TTS（读出判词或战报标题）

### 20.6 音频触发点（必须明确写入交互）
- 进入 Hub：Hub BGM + 雨声 ambience
- 触发镜子：镜面“低频轰鸣”+ 裂纹 SFX
- 点击抽屉：对应模块切 BGM（淡入淡出 300–600ms）
- 点选二选一：stinger（轻/重两种）
- 获得碎片：对应碎片 SFX（红豆=轻叮、糖=纸包、蹄印=闷响）
- 生成分享卡：打印机/快门 SFX + 一句短配音（可选）

## 21. 用 AI 做“内容更有趣”的方法（把大模型当导演，不当写手）

### 21.1 内容生成要“可控 + 可复现 + 可变体”
不要让模型自由写长文，而是让它完成“受控任务”：
- 生成 1 段叙事（固定长度）
- 生成 2 个选项（固定格式）
- 生成 1 句可截图短句（固定上限）
- 生成 1 个音频指令（BGM/Voice/SFX 的选择，不让它胡编路径）

### 21.2 叙事的“多层结构模板”（更耐玩）
每段 narration 建议固定 4 句结构（模型不易跑偏，效果更像剧本）：
1) 现实钩子（工位/Hackathon/雨）
2) 超现实转译（镜子/召唤师/深夜食堂）
3) 行为结果（你做了什么，世界怎么变）
4) 情绪落点（给一个短而准的句子）

### 21.3 结局多样化：从“3 个结局”升级到“结局谱系”
把结局定义为“动作×语气×代价”，自然爆出很多变体：
- 动作：寄出/塞回/留半价/只带走/逃亡/称王
- 语气：温柔/讽刺/荒诞/冷静
- 代价：裂纹上升/碎片消耗/充能变化

同一个“寄出”，也能生成完全不同的“寄出方式”：邮局/树洞/镜子背面/服务区的信箱/马背上的口袋，极其丰富。

### 21.4 关键：把玩家输入“二次加工”为可玩素材
玩家输入不是直接进大模型写故事，而是先被系统提炼成“可玩素材”：
- 抽取：情绪标签（焦虑/委屈/想逃/想证明）与关键词（人物/地点/物品）
- 转译：生成一个“镜子译文”（更中二/更诗意/更抽象，取决于 fracture）
- 再交给模型：让它围绕译文生成回合内容

这样用户即使输入很短，也能产出丰富内容。

## 22. JSON 协议增强：把音频直接纳入模型输出（落地更快）

### 22.1 协议新增 audio 字段（模型只“选”，不“编”）
建议在 9.1 结构上新增：
```json
{
  "audio": {
    "bgm": "hub|bar|letter|road|final",
    "ambience": ["rain","keyboard","neon","traffic"],
    "sfx": ["mirror_crack","drawer_open","stamp","hoof","crown"],
    "voice": [
      { "role": "mirror|cat|director|classmate", "text": "string", "style": "calm|warm|sharp|epic" }
    ]
  }
}
```

约束：
- bgm/ambience/sfx 必须从预置枚举里选择
- voice.text 控制在 8–18 字（短句），适合 TTS

### 22.2 Prompt 增强要点（每个模块加一条）
在各模块 Prompt 的“输出要求”中补充：
- 必须输出 audio 字段
- BGM 只能选对应模块
- voice.role 固定为本模块角色（猫/镜子/导演/同桌）
- voice.text 必须可独立成句，适合配音

这样你们前端/平台侧只要按枚举播放音频，就能立刻拥有“配乐配音”的沉浸感。

## 23. 音频枚举与素材清单（可直接交给负责音频/前端同学）

### 23.1 BGM 枚举（9 条循环）
命名建议：`bgm_<key>_v1.mp3`
- hub：工位夜雨（低频、轻键盘节奏）
- bar：深夜食堂 Lo-fi（暖、爵士和弦）
- letter：未寄出回信（钢琴/木吉他、留白）
- road：公路逃亡（合成器驱动、鼓点）
- final：逆风终局（上扬但克制）
- hub_fracture_high：hub 的荒诞变体（不稳定噪声/错位拍点）
- hub_fracture_low：hub 的克制变体（更少鼓点、更多留白）
- stinger_crown：加冕短音（2–4 秒）
- stinger_guyu：谷雨破局短音（2–4 秒，“雨停/开阔”）

### 23.2 Ambience 枚举（4 条循环）
命名建议：`amb_<key>_v1.wav`
- rain：雨声（谷雨锚点）
- keyboard：键盘/风扇/室内电流
- neon：霓虹电流/电梯嗡鸣
- traffic：远处车流/公路风噪

### 23.3 SFX 枚举（6 条短音）
命名建议：`sfx_<key>_v1.wav`
- mirror_crack：镜面裂纹
- drawer_open：抽屉打开
- stamp：盖章/打印机出纸（用于“寄出/分享卡”）
- hoof：马蹄/落地闷响
- crown：加冕短音
- shutter：快门/截图确认（分享卡生成）

### 23.4 Voice 角色枚举（4 个）
命名建议：`tts_<role>_<id>_v1.wav`（缓存版）
- mirror：镜子
- cat：猫酒保
- director：召唤师导演
- classmate：同桌回信（或“回信的声音”）

## 24. AI 生成音频的提示词（可直接复制到任意 AI 作曲/TTS 平台）

目标：生成“原创、可循环、无明显旋律引用、无歌词/无人声”的可用资产。

### 24.1 通用约束（所有 BGM 都加在提示词末尾）
- 只生成原创音乐，不模仿任何已知作品与旋律
- 不要人声，不要可辨识语言
- 输出 30–45 秒可无缝循环版本（loopable）
- 频段干净，不要过度压缩，不要刺耳高频
- 适合背景播放，不抢文本叙事

### 24.2 9 条 BGM 的提示词（逐条生成）

1) hub（工位夜雨）
```
风格：夜晚办公室 / 轻微赛博感的 lo-fi
情绪：疲惫但还在坚持，轻微希望
节奏：72–84 BPM，稳、不过分律动
配器：低频合成器垫底 + 轻鼓点 + 细碎键盘打字节奏（可用打击乐模拟）+ 少量电钢琴和弦
空间：小房间混响，贴耳
结构：A段(15s) + 轻变化(15s) + 回到A(15s)，可循环
```

2) bar（深夜食堂）
```
风格：暖 lo-fi / 轻爵士
情绪：被理解、被照顾的松弛
节奏：78–92 BPM，摆动感微弱
配器：电钢琴和弦 + 轻贝斯 + brush鼓/轻军鼓 + 少量萨克斯/口琴“点缀”（不要主旋律）
结构：舒缓循环，结尾不要收束
```

3) letter（未寄出回信）
```
风格：极简钢琴 / 木吉他氛围
情绪：克制的意难平，留白
节奏：自由节拍或极弱拍点（可听不出鼓）
配器：钢琴断句 + 远处的Pad + 偶尔的指弹吉他泛音
结构：不推进、不高潮，像在写信
```

4) road（公路逃亡）
```
风格：合成器驱动的公路片
情绪：逃离、速度、荒诞爽感
节奏：100–118 BPM，鼓点更明确
配器：合成器bass + 电子鼓 + 简单重复动机（不要像流行歌主旋律）+ 风噪质感纹理
结构：每 8 小节轻微变奏，可循环
```

5) final（逆风终局）
```
风格：克制的燃 / 决心感
情绪：不是胜利，而是“我终于选择了”
节奏：90–105 BPM，推进但不炸
配器：弦乐Pad/合成器弦 + 稳定底鼓 + 低频持续音 + 轻微上行和声
结构：上扬但保持留白，可循环
```

6) hub_fracture_high（裂纹高的 hub）
```
基于 hub 的氛围做“故障化变体”
情绪：荒诞、抽象、略不安
节奏：保持原BPM，但加入错位拍点与轻微失真
配器：增加颗粒噪声、断裂的电流音色、短促反拍
结构：可循环，不要变成恐怖片
```

7) hub_fracture_low（裂纹低的 hub）
```
基于 hub 的氛围做“更克制的变体”
情绪：安静、清醒、像雨后
节奏：拍点更少，更多留白
配器：减少鼓点，增加柔软的电钢琴与低频Pad
结构：可循环
```

8) stinger_crown（加冕短音）
```
风格：短促胜利提示音
时长：2–4 秒
情绪：爽、但不俗气
配器：低频击打 + 一次上行和声 + 很短的尾音
```

9) stinger_guyu（谷雨破局短音）
```
风格：雨停后的开阔提示音
时长：2–4 秒
情绪：松一口气、世界变亮
配器：清脆高音点亮 + 轻柔和弦落地
```

### 24.3 SFX 的 AI 生成提示词（如需要）
如果不用素材库，可以用 AI 生成短音效（0.2–1.2 秒）：
- mirror_crack：玻璃细裂 + 低频震动（不尖锐）
- drawer_open：木质抽屉拉开 + 轻碰撞
- stamp：纸张出纸 + “啪”一声盖章
- hoof：马蹄落地闷响（偏低频）
- crown：金属轻碰 + 亮一下
- shutter：快门咔嚓 + 轻提示音

## 25. TTS 配音方案（声线描述 + 台词生成方式）

### 25.1 四个角色的声线描述（给任意 TTS 平台）
镜子 mirror：
- 性格：冷静、像系统日志、少情绪但很准
- 声线：低频、中速、停顿明确
- 表达：短句、反问多，不撒鸡汤

猫 cat：
- 性格：嘴毒但温柔，吐槽感强
- 声线：中频略快，句尾带轻微“哼/收住”的感觉
- 表达：先刺一句，再给一条能用的安慰

导演 director：
- 性格：夸张播报，像在解说你的人生跑酷
- 声线：中高频、节奏强、断句像鼓点
- 表达：短句冲击力强，适合战报标题

同桌 classmate：
- 性格：克制、留白，像在夜里念信
- 声线：轻声、中慢速、尾音短
- 表达：不解释太多，只点到为止

### 25.2 “缓存台词库”制作方法（保证上线不卡）
建议先做两类短句缓存（各 30–60 条即可）：
- 镜子判词（每局结算播一句）
- 猫金句（模块一播一句）
做法：用 TTS 平台批量合成，按 `tts_role_id_v1.wav` 命名缓存。

### 25.3 动态 TTS（可选增强）
实时生成只做 1 句“摘要”，避免卡顿：
- 模块一：把用户烦恼提炼成 8–12 字摘要，猫读出来
- 模块三：战报标题 8–12 字，导演读出来
